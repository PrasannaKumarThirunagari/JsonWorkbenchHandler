Act as a Senior UI/UX Designer and Frontend Architect specializing in developer tools, data visualization, and JSON intelligence platforms.

🎯 Context:
This application is a JSON Intelligence tool used for viewing, analyzing, editing, validating, and transforming JSON data. The users are primarily developers, analysts, and technical users.

🔍 Phase 1: Product & User Understanding

* Analyze how users interact with JSON (viewing, editing, debugging, comparing, transforming)
* Identify key workflows such as:

  * JSON viewer & explorer
  * Schema validation
  * API response inspection
  * Search, filter, and navigation within large JSON
  * Diff comparison between JSON objects

🧠 Phase 2: UX Audit (Data-Centric)

* Identify friction in handling large and nested JSON structures
* Evaluate readability, indentation, and hierarchy clarity
* Detect issues in navigation (expand/collapse, deep nesting)
* Identify missing power-user features (keyboard shortcuts, quick actions)

🎨 Phase 3: Design System (Developer-Friendly)

* Create a clean, minimal, distraction-free UI theme (light & dark mode)
* Typography optimized for code readability (monospace + UI font pairing)
* Consistent spacing for structured data visualization
* Color system for syntax highlighting (keys, values, types, errors)

🧩 Phase 4: Core UI Components (Specialized)
Design and improve:

* JSON Tree Viewer (expand/collapse, lazy loading for large data)
* Code Editor (syntax highlighting, formatting, linting)
* Side-by-side JSON Diff Viewer
* Search & filter panel (key/value/type-based search)
* Breadcrumb navigation for deep nested paths
* Schema validator & error highlighting panel
* Copy, download, and transform actions

✨ Phase 5: Advanced Features (Critical)

* Smart JSON insights (auto-detect structure, highlight anomalies)
* Inline editing with validation feedback
* Collapse/expand all, level-based expansion
* JSON path generator (auto-create paths on selection)
* Data type indicators (string, number, boolean, null)
* Performance handling for large JSON (virtual scrolling)
* History / version comparison

🧭 Phase 6: Navigation & Layout

* Split layout (Explorer | Editor | Insights panel)
* Resizable panels for better control
* Sticky headers and contextual toolbars
* Quick access actions (format, validate, beautify, minify)

📱 Phase 7: Responsiveness & Performance

* Desktop-first design (primary users are developers)
* Optimize rendering for large JSON datasets
* Use virtualization and efficient DOM updates

⚙️ Phase 8: Output Deliverables
Provide:

* Complete UI layout structure (panel-wise and screen-wise)
* Design system (colors, typography, spacing, syntax theme)
* Component-level UX improvements
* Feature recommendations with reasoning
* TailwindCSS-based implementation suggestions (if applicable)

🚀 Final Goal:
Deliver a professional, high-performance JSON Intelligence UI that is:

* Extremely readable and efficient for large data
* Optimized for developer productivity
* Clean, minimal, and distraction-free
* Comparable to tools like Postman, VS Code JSON viewer, or Swagger UI
